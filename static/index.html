The Universal Canvas

By Milo Kinkead

What am I looking at?

The Universal Canvas is infinitely detailed and contains every finite pattern. This means that any possible image of any resolution can be found within the Universal Canvas, simply by zooming in.

It goes beyond that, though. Not only does every image exist within the Universal Canvas, every image exists within every subdivision of the Canvas. So, if you cut a section out of the canvas, no matter how small, that section will also contain every possible image.

What does this actually mean?

In the first section, I glossed over how a 'pattern' or an 'image' is actually defined. If an image was just a grid of pixels, each having a perfect solid color, then what I said about every subdivision containing every image would obviously be impossible, as the subdivision of a single black pixel would contain nothing but black.

The way that I define an image is as a grid of pixels, each of which has not a solid color, but an average color. This way, a rendered image can have the appearance of solid colors due to the averaging of details smaller than a pixel, while still having infinite detail within each pixel. So, an image 'existing' at a given resolution really means that there exists some region within the Universal Canvas, that, if rendered at said resolution by averaging details smaller than a pixel, would produce the image.

How is it implemented?

So, I've gone on about how the Universal Canvas has infinite detail and contains infinite information, but at the end of the day there needs to be some way to simulate it without actually doing infinite computation and storing infinite data.
